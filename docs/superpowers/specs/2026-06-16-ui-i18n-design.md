# UI Korean/English Toggle Design

## Status

Approved for implementation planning.

## Goal

Add a maintainable Korean/English UI language toggle without changing forensic
evidence values, artifact names, rule identifiers, paths, timestamps, or backend
payload semantics. The first implementation should make the daily analyst
workflow usable in Korean while preserving English as the default language.

## Scope

The first pass covers high-traffic UI chrome and analyst workflows:

- Sidebar navigation and section labels.
- Header metadata labels, status text, buttons, and keyboard shortcut popover.
- Dashboard summary labels and status copy.
- Artifact Browser search controls, pagination/status text, detail panel labels,
  and empty-result guidance copy.
- Timeline filter labels and status text.
- Settings and dependency-status labels that affect readiness interpretation.

The first pass intentionally does not translate:

- Evidence values, file paths, hostnames, usernames, hashes, IOCs, event fields,
  registry keys, command lines, or raw artifact contents.
- Artifact family names returned by the backend.
- MCP tool names, rule ids, MITRE technique ids, and generated report evidence
  sections.
- Backend error strings, unless they are wrapped by a frontend-owned label.

## Architecture

Add a small frontend-only i18n layer under `frontend/src/i18n/`:

- `translations.ts`: typed dictionaries for `en` and `ko`.
- `useI18n.ts`: hook that exposes `language`, `setLanguage`, and `t(key)`.
- `keys.ts` or a typed key export if the dictionary grows enough to need it.

Use the existing Zustand store for language state:

- `language: 'en' | 'ko'`.
- `setLanguage(language)`.
- Initialize from `localStorage['language']`, falling back to `en`.
- Persist language changes back to `localStorage`.

The translation function returns the current language value when present and
falls back to English. Missing keys must not break rendering.

## UI Behavior

Add a compact `EN / KO` segmented control in the header near the theme/MCP
controls. The current language is visually active. Switching language updates
the UI immediately and persists across reloads.

Keep technical acronyms such as MCP, DFIR, IOC, EVTX, KAPE, MFDB, YARA, Ghidra,
SRUM, and MITRE unchanged in both languages unless a surrounding label needs
Korean grammar.

Korean labels must be concise enough for the existing dense operational layout.
Where Korean text is longer than the English label, prefer:

- short labels in controls;
- longer explanatory text only in status/help areas;
- ellipsis for bounded metadata values.

## Data Flow

Components import `useI18n()` and replace hard-coded UI strings with `t(...)`.
Dynamic backend data remains unchanged. For mixed strings, components should
translate only frontend-owned static fragments and interpolate raw values
unchanged.

Example:

```tsx
const { t } = useI18n();
<Meta label={t('header.case')} value={activeCaseName} />
```

## Error Handling

If a translation key is missing in Korean, render the English string. If it is
missing in both dictionaries, render the key name in development-friendly form
so the issue is visible but the UI remains usable.

Backend error payloads stay as-is. Frontend wrappers such as "Load failed" or
"No rows" can be translated.

## Testing

Run the frontend build after implementation:

```powershell
npm.cmd run build
```

Add a small TypeScript-level consistency check only if the existing frontend
tooling supports it without introducing a new test framework. At minimum,
typing the dictionaries with a shared key shape should catch missing Korean
keys during build.

Manual verification checklist:

- Header language toggle switches visible labels immediately.
- Reload preserves the selected language.
- Sidebar labels fit without layout breakage.
- Artifact Browser search, pagination, zero-result hint, and detail panel remain
  usable in Korean.
- Timeline and Dashboard labels do not overlap in Korean.
- Evidence values and backend-provided artifact/rule names remain untranslated.

## Risks

- The project has many hard-coded strings, so a complete translation in one pass
  can create broad churn. Mitigation: prioritize shared chrome and core
  investigation workflows first.
- Korean labels are often longer. Mitigation: use concise labels and existing
  truncation patterns rather than widening global layout.
- Translating forensic data can harm reproducibility. Mitigation: translate only
  frontend-owned UI text and leave evidence payloads unchanged.
