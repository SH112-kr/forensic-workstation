# UI Korean/English Toggle Implementation Plan

Goal: Add a persistent Korean/English UI language toggle and translate the main analyst workflow without translating evidence data.

## Scope

- Add frontend-only i18n dictionaries for English and Korean.
- Persist the selected language in the existing Zustand store.
- Add an EN/KO segmented toggle in the header.
- Translate UI-owned labels, buttons, empty states, and workflow help text in the main analyst screens.
- Keep backend-provided evidence values raw: artifact family names, rule ids, hashes, paths, commands, registry keys, timestamps, and tool output.
